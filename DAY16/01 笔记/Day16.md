# 事件高级

### H5注册事件方式

```
传统注册方式
    1,利用o开头的事件来注册，本质是设置元素的属性
    2,同一个元素的同一事件，只能设置一个处理函数，后面注册的函数会覆盖前面的
H5监听注册方式
    1,利用addEventListener()注册，本质是调用法
    2,同一个元素的同一事件可以设置多个处理函数，按照注册顺序依次执行
```

### DOM事件流

```
DOM事件流
	事件发生时会在元素节点之间按照特定的顺序传播，这个传播过程就是事件流
事件捕获
	网景公司最早提出的，事件开始是由DOM最顶层的节点接收，然后逐级向下传播到最具体的元素

事件冒泡  
    IE最早提出的，事件开始是由最具体的元素接收，然后逐级向上传播到DOM最顶层节点的过程

w3C统一标准---事件先捕获再冒泡
	
	DOM事件流的3个阶段：捕获阶段---当前目标阶段（最具体元素）--冒泡阶段  

addEventListener()
	第三个参数如果是tru,表示在事件捕获阶段调用事件处理函数
	
	如果是fa1se(如果不写，默认就是fa1se),表示在事件冒泡阶段调用事件处理函数
	
	传统注册事件方法onc1ick只能得到冒泡阶段
    
    实际开发中，我们很少使用事件捕获，更关注事件冒泡
    
    有些事件是没有冒泡的，比如onblur,onfocus,onmouseenter,onmouseleave
```



### 事件对象

```
概念
    事件发生之后，跟事件相关的一系列的信息数据的集合，都会放在这个对象中，这个对象就是事件对象
    比如
        1,随绑定了这个事件
        2,如果是鼠标触发的事件，可以得到鼠标相关的信息，比如鼠标的位置
        2,如果是键盘触发的事件，可以得到键盘相关的信息，比如按下了哪个键
获取
    事件在触发时候，就会产生事件对象，并且系统会以实参的形式传递给事件处理函数
    我们可以在事件外理函数中声明一个参数来接收事件对象
    
作用
    1,可以获取当前事件的类型(e.type)
    2,在键盘事件中可以监听按下了哪一个键(e.keyCode)
    3,可以获取具体点击了哪一个子元素(e.target)
    4,可以阻止某些标签的默认行为(e.preventDefault))
    5,可以阻止事件冒泡(e.stopPropagation())	    

```
