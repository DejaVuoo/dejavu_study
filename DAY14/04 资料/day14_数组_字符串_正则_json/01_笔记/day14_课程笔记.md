#### 正则对象

```
概念
	正则表达式是描述字符模式的对象。
	正则表达式用于对字符串模式匹配及检索替换，是对字符串执行模式匹配的强大工具。
	
	写一个正则校验手机号  ^1[35789][0-9]{9}$  ^1[35789]\d{9}$
		1,第一位以1开头
		2,第二位是3,5,7,8,9
		3,总共有11位
		4,全部都是数字
		
	写一个校验qq号 ^[1-9][0-9]{4,14}$
	
规则	
	单字符 代表它自己
	
	[]方括号用于查找某个范围内的字符
	[12345] 表示1-5中的任意一个
	[0-9] 表示0-9中的任意一个
	[^0-9] 表示非数字(相当于上面取反)
	[a-zA-Z] 表示英文字符
	[^a-zA-Z] 表示非英文字符 
	
	数量词:表示前面字符出现的次数
	[0-9]+  +表示至少一次
	[0-9]*  *表示任意次(0次,1次或者多次)
	[0-9]?  ?表示0次或者1次
	[0-9]{x}  表示出现x次
	[0-9]{x,}  表示至少出现x次
	[0-9]{x,y}  表示至少出现x次,最多出现y次
	
	//元字符:类似关键字,有特殊含义
	\d 表示0-9的数字  和[0-9]的含义是一样的
	\D 表示非数字 和[^0-9]的含义是一样的
	\w 表示单词字符,和[a-zA-Z0-9]含义是一样的 (小写的w)
	\W 表示单词字符,和[^a-zA-Z0-9]含义是一样的 (大写的w)
	
	^以xx开头
	$以xx结尾
```



#### Json格式

```
概念
	JavaScript Object Notation(JavaScript 对象表示法)
	JSON 是存储和交换文本信息的语法，类似 XML。
	一种轻量级的数据交换格式,本质是一个字符串
	
特点
	JSON 比 XML 更小、更快，更易解析。
	JSON 易于人阅读和编写。
	
作用
	主要用于存储和传输数据,一般用于服务端向客户端传递数据

区别
	xml是一个重量级的语法,注意用于后台存储数据(标签占用的提及比较大)
        <user>
            <username>张三</username>
            <password>123</password>
        </user>
        
	json是一个轻量级的语法,主要用于网络间数据传输,占用的体积比较小
		{"username":"张三","password":"123"}
		
转换
	JSON.parse()	用于将一个 JSON 字符串转换为 JavaScript 对象。
    JSON.stringify()	用于将 JavaScript 值转换为 JSON 字符串。
```
